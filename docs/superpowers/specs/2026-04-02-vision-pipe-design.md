# vision-pipe: Зрительная система для AI-агентов

**Дата:** 2026-04-02
**Статус:** Draft

## Проблема

AI-агенты (OpenClaw и др.) воспринимают экран через снапшоты: скриншот → отправка в LLM → ожидание ответа (~30 секунд на страницу). Это медленно, блокирует работу агента и не работает на площадках, которые блокируют API-ботов.

Человек смотрит на страницу и за 1-2 секунды понимает что на ней. vision-pipe даёт агентам такое же быстрое зрение.

## Решение

MCP-сервер, который непрерывно наблюдает за экраном и предоставляет агенту семантическое понимание — не скриншоты, а осмысленные описания, как зрительная кора передаёт информацию мозгу.

## Ключевые принципы

- **Глаза, не камера** — vision-pipe возвращает агенту обработанную информацию (текстовые описания, структурированные данные), а не сырые изображения
- **Скорость восприятия** — главная метрика. Цель: 2-3 секунды на понимание страницы вместо 30
- **Pluggable провайдеры** — любая VLM-модель подключается через единый интерфейс
- **Фокус внимания** — агент управляет тем, на что vision-pipe обращает внимание

## Архитектура

Двухфазная модель, вдохновлённая человеческим зрением (периферийное + фовеальное):

```
Экран (macOS)
    │
    │ захват (ScreenCaptureKit)
    ▼
┌─────────────────────────────────────┐
│  ДЕТЕКТОР ИЗМЕНЕНИЙ (pixel diff)    │
│  "Экран изменился? Где именно?"     │
│  Стоимость: ~0 (чистая математика)  │
└──────────────┬──────────────────────┘
               │ только при изменении
               ▼
┌─────────────────────────────────────┐
│  ФАЗА 1: ПЕРИФЕРИЙНОЕ ЗРЕНИЕ       │
│  Быстрая лёгкая модель (pluggable) │
│  Вход: скриншот низкого разрешения  │
│  Выход: карта регионов + описание   │
│  Скорость: ~0.3-1s                  │
└──────────────┬──────────────────────┘
               │ карта регионов
               ▼
┌─────────────────────────────────────┐
│  ФАЗА 2: ФОВЕАЛЬНЫЙ ФОКУС          │
│  Мощная модель (pluggable)          │
│  Вход: кроп нужного региона (hi-res)│
│  Выход: детальное понимание         │
│  Скорость: ~1-2s                    │
└──────────────┬──────────────────────┘
               │ семантическое описание
               ▼
┌─────────────────────────────────────┐
│  МОДЕЛЬ МИРА (World Model)          │
│  Текущее понимание экрана           │
│  Обновляется инкрементально         │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  MCP СЕРВЕР                         │
│  Tools + Notifications → Агент      │
└─────────────────────────────────────┘
```

### Детектор изменений

Pixel diff между текущим и предыдущим кадром. Чистая математика, нулевая стоимость. Если `sum(abs(frame_t - frame_t-1)) < threshold` — ничего не происходит, модели не вызываются. Определяет не только факт изменения, но и область изменения на экране.

### Фаза 1: Периферийное зрение

Быстрая модель получает скриншот в низком разрешении (512x512) и за ~0.3-1 секунду отвечает:
- Что это за приложение/страница
- Какие регионы на экране (header, content, sidebar, terminal и т.д.)
- Где находится потенциально релевантный контент

Не нужно детальное понимание — только карта регионов и общее описание.

### Фаза 2: Фовеальный фокус

Мощная модель получает кроп конкретного региона в полном разрешении и за ~1-2 секунды извлекает:
- Детальное содержимое области
- Структурированные данные (температура, цены, ошибки и т.д.)
- Визуальные элементы (графики, дизайн, изображения)

Вызывается только для нужных регионов — не для всей страницы.

### Модель мира (World Model)

Постоянно поддерживаемое понимание текущего состояния экрана. Обновляется инкрементально — при изменении пересчитывается только изменившийся регион.

```
WorldModel {
  summary: "VS Code с файлом main.py, терминал внизу"

  regions: [
    {
      name: "editor"
      bounds: {x, y, w, h}
      description: "Python код, функция parse_data"
      focus: "normal"
      last_change: "14:03:12"
    },
    {
      name: "terminal"
      bounds: {x, y, w, h}
      description: "Запущен pytest, 3 passed"
      focus: "high"          # агент навёл фокус
      last_change: "14:03:45"
    }
  ]

  history: [
    {time: "14:03:45", delta: "Терминал: pytest завершился, 3 passed"},
    {time: "14:03:12", delta: "Редактор: курсор перемещён на строку 42"},
  ]
}
```

Агент получает от vision-pipe не скриншоты, а эту модель мира — как мозг получает обработанные сигналы от зрительной коры.

## MCP-интерфейс

### Tools

**Восприятие:**

| Tool | Описание | Пример ответа |
|------|----------|---------------|
| `get_state()` | Текущая модель мира | `"VS Code, файл main.py:42, терминал idle"` |
| `describe(region?)` | Детальное описание области (запускает фовеальный фокус). Region — по имени из модели мира (`"terminal"`) или по координатам (`{x,y,w,h}`). Без аргумента — весь экран. | `"Температура в стране A: 23°C, облачно"` |
| `get_changes(since?)` | Что изменилось с последнего запроса | `[{time, change: "Терминал: ошибка ImportError"}]` |

**Управление фокусом:**

| Tool | Описание | Эффект |
|------|----------|--------|
| `focus(region, priority?)` | Следить за областью внимательнее | Понижает порог срабатывания для региона |
| `ignore(region)` | Игнорировать область | Реклама, статичные элементы |

**Конфигурация:**

| Tool | Описание |
|------|----------|
| `set_provider(phase, provider)` | Сменить модель на лету |

### Notifications (push)

| Notification | Когда срабатывает |
|-------------|-------------------|
| `vision/change` | Значимое изменение на экране |
| `vision/focus_alert` | Изменение в области с высоким фокусом |

### Пример диалога

```
АГЕНТ: открывает weather.com через browser_navigate
  ... страница загружается ...

VISION → АГЕНТ: vision/change
  "Браузер: загрузился weather.com, сайт погоды,
   регионы: поиск, карта, текущая погода, прогноз"

АГЕНТ: describe("текущая погода")
  → "Страна A: 23°C, облачно, влажность 67%,
     ветер 15 км/ч СЗ, ощущается как 21°C"

АГЕНТ: переходит на следующий сайт
  ... 2 секунды на всё ...
```

## Система провайдеров

### Три типа провайдеров

**Sampling (через агента):**
Vision-pipe использует MCP Sampling — просит агента обработать изображение через его собственную модель. Без API-ключей, работает из коробки. Можно передать hint на предпочитаемую модель (haiku для скорости, sonnet для качества).

**Локальные (on-device):**
Модели запускаются на машине пользователя — Ollama, MLX (Apple Silicon), Apple Vision Framework. Полный контроль и приватность.

**Облачные (по API-ключу):**
Прямые вызовы к API: Gemini, Claude, GPT-4o и другие. Максимальное качество, не нагружают машину.

Все типы свободно миксуются: периферия через sampling/haiku, фовеа через gemini-flash — или любая другая комбинация.

### Интерфейс провайдера

```python
class VisionProvider(Protocol):
    async def scan(self, image: bytes) -> ScanResult:
        """Фаза 1: быстрый scan всего экрана."""
        ...

    async def focus(self, image: bytes, region: Region,
                    context: str) -> FocusResult:
        """Фаза 2: детальный анализ региона."""
        ...
```

### Кастомные провайдеры

Три способа добавить провайдер:

1. **Встроенные** — идут из коробки (ollama, gemini, claude, apple-vision, mlx, openai)
2. **Pip-пакеты** — `pip install vision-pipe-provider-X`, автообнаружение через Python entry points
3. **Локальный файл** — `custom:./path/to/provider.py` для экспериментов

Entry point в `pyproject.toml` пакета провайдера:

```toml
[project.entry-points."vision_pipe.providers"]
replicate = "vision_pipe_provider_replicate:ReplicateProvider"
```

Команда `vision-pipe providers` автоматически находит все установленные провайдеры.

### Конфигурация

```yaml
# vision-pipe.yaml

peripheral:
  provider: ollama/moondream2
  resolution: 512x512        # низкое для скорости

foveal:
  provider: gemini-flash
  resolution: original        # полное для точности

capture:
  fps: 4                      # проверка экрана 4 раза в секунду
  diff_threshold: 5%          # минимум изменений для триггера
```

Готовые пресеты:

```bash
vision-pipe serve --preset fast       # скорость > качество
vision-pipe serve --preset balanced   # баланс
vision-pipe serve --preset quality    # качество > скорость
```

## Benchmark Suite

Набор эталонных скриншотов с известными правильными ответами для тестирования качества восприятия.

### Эталонные скриншоты

Покрытие разных типов приложений:

| Категория | Скриншоты |
|-----------|-----------|
| Браузер | weather.com, GitHub, Gmail, Google Docs, Twitter/X, YouTube, Amazon, Stack Overflow |
| Редакторы | VS Code (код), VS Code (ошибка), Xcode, Cursor |
| Терминал | build output, htop, logs, interactive CLI |
| Дизайн | Figma, Sketch, Preview с изображением |
| Коммуникации | Slack, Telegram, Discord |
| Продуктивность | Notion, Excel/Numbers, Calendar, Finder |
| Мультимедиа | Spotify, QuickTime, фото в Preview |
| Сложные | мульти-окна, split screen, уведомления поверх контента |

### Метрики

**Фаза 1 (периферийное зрение):**
- Latency — время до первого ответа
- Region accuracy — правильно ли определены регионы
- Relevance — нашла ли важные области

**Фаза 2 (фовеальный фокус):**
- Latency — время детального анализа
- Extraction accuracy — точность извлечённых данных
- Completeness — всё ли важное найдено

### CLI

```bash
# Бенчмарк одного провайдера
vision-pipe benchmark --provider ollama/llava:7b

# Сравнение всех доступных провайдеров
vision-pipe benchmark --compare all

# Бенчмарк только фазы 1
vision-pipe benchmark --phase peripheral --compare all
```

## Платформа

- **macOS only** для первой версии
- Захват экрана через ScreenCaptureKit (macOS 12.3+)
- Python 3.11+
- Распространение через PyPI: `pip install vision-pipe`

## Подключение к агенту

```json
{
  "mcpServers": {
    "vision": {
      "command": "vision-pipe",
      "args": ["serve", "--preset", "fast"]
    }
  }
}
```

Одна строка в конфиге — и у агента есть зрение.
